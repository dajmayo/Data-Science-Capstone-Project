Data Cleaning/Preprocessing

Converting to corpus objects.

# Combine the text documents and convert to corpus objects
master_vector <- c(s_blog, s_news, s_twitter)
corp <- corpus(master_vector)

We now remove the unnecessary items in the text files not needed in predicting.

# Removing the punctuations and/or numbers
cleaning <- tokens(
    x = tolower(corp),
    remove_punct = TRUE,
    remove_twitter = TRUE,
    remove_numbers = TRUE,
    remove_hyphens = TRUE,
    remove_symbols = TRUE,
    remove_url = TRUE
)
# After cleaning, we now stem it
stem_words <- tokens_wordstem(cleaning, language = "english")

Tokenization and N-Gram Modeling

Lets create N-grams and Document Frequency Matrices from the corpus.

bi_gram <- tokens_ngrams(stem_words, n = 2)
tri_gram <- tokens_ngrams(stem_words, n = 3)

uni_DFM <- dfm(stem_words)
bi_DFM <- dfm(bi_gram)
tri_DFM <- dfm(tri_gram)

# Let us trim the N-Grams for faster calculations
uni_DFM <- dfm_trim(uni_DFM, 3)
bi_DFM <- dfm_trim(bi_DFM, 3)
tri_DFM <- dfm_trim(tri_DFM, 3)

Now lets group the word tokens to find the count in the corpus.

# Create named vectors with counts of words 
sum_uni <- colSums(uni_DFM)
sum_bi <- colSums(bi_DFM)
sum_tri <- colSums(tri_DFM)

# Create data tables with individual words as columns
uni_words <- data.table(One_Word = names(sum_uni), Frequency = sum_uni)

bi_words <- data.table(
        One_Word = sapply(strsplit(names(sum_bi), "_", fixed = TRUE), '[[', 1),
        Two_Words = sapply(strsplit(names(sum_bi), "_", fixed = TRUE), '[[', 2),
        Frequency = sum_bi)

tri_words <- data.table(
        One_Word = sapply(strsplit(names(sum_tri), "_", fixed = TRUE), '[[', 1),
        Two_Words = sapply(strsplit(names(sum_tri), "_", fixed = TRUE), '[[', 2),
        Three_Words = sapply(strsplit(names(sum_tri), "_", fixed = TRUE), '[[', 3),
        Frequency = sum_tri)

Next Steps: Prediction Algorithm and Shiny Application

Now that were done with our exploratory data analysis, the next step is to create a predictive model for word prediction and deploy it into a shiny application so external users are able to benefit from it.

How to make it?

  • Preprocessing executed should be considered
  • Using the n-gram model with frequency created above.
  • Feasibly, we will be using the trigram model to predict the next word. If no match will be found, the algorithm would most likely back off to the bigram model, them to the unigram model if necessary.

How will it look like?

The user interface of the shiny application is expected to be as simple and user-friendly as possible for the ease of the users in navigating. Lastly, hopefully that the application would be able to accurately and quickly help the users for word prediction.
